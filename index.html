Deployment

Up and running in 60 seconds.

# docker-compose.yml
version: '3'
services:
  runner_arm:
    image: neytor/runner_arm:latest
    container_name: runner_arm
    restart: always
    environment:
      - runner_token=YOUR_TOKEN         # required
      - runner_url=https://github.com/YOUR_ORG  # required
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    privileged: true
docker container run \
  --name runner_arm \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -e runner_url=https://github.com/YOUR_ORG \
  -e runner_token=YOUR_TOKEN \
  --privileged \
  -d neytor/runner_arm:latest

Generate the token at: Settings → Actions → Runners → New self-hosted runner → Configure section.

Configuration

Environment variables.

Variable Required Default Description
runner_token required — Registration token from GitHub (Settings → Actions → Runners)
runner_url required — URL of the organization or repository to register against
runner_user optional runner OS user that owns and runs the agent process
runner_home_dir optional /home/runner Home directory for the runner user

Automation

Self-updating CI pipeline.

Every Monday at 08:00 UTC the workflow wakes up, checks the latest actions/runner release, and rebuilds the image only when a new version is available.

PHASE 1
🔍
Version check
Queries GitHub API for the latest runner release and compares it to the current Docker Hub tag.
PHASE 2
🏗️
Native build
Builds on ubuntu-24.04-arm — a real ARM64 runner, no emulation.
PHASE 3
🚀
Push & tag
Pushes latest and the exact version tag to Docker Hub.
PHASE 4
📋
GitHub Release
Creates a versioned GitHub Release with quick-start instructions and changelog link.
